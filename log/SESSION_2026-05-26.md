## Problem Areas

1. **s-expr-builder outputs flat single-line s-expressions** — Nested `sexpr()` calls return strings (not `SExprNode` objects), so the builder treats them as simple children and puts everything on one line. The output is still valid s-expressions (KiCad 9 should parse it), but it's hard to read. Fix would require making `sexpr()` return `SExprNode` objects with a separate formatting pass, which touches every call site.

2. **`fc()` precision** — Uses 4 decimal places (`toFixed(4)`) unconditionally, producing values like `101.6000` instead of `101.6`. This is valid but doesn't match KiCad's own output formatting exactly.

3. **Copper zone `filled_polygon` is approximated** — Generated as a simple 4-point inset rectangle rather than KiCad's exact corner-rounded offset polygon. Should still load correctly but won't match KiCad's exact fill geometry.

4. **No `In1.Cu`/`In2.Cu` copper layers** — Example2 uses a 4-layer board with inner signal layers. Our export only has F.Cu and B.Cu. This is fine for the simple 2-layer case but won't match if user needs inner layers.
