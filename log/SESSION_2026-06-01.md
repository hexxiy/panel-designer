### Text Tool

Two iterations:

1. **First attempt — text-as-part:** Created `silkscreen_text` / `copper_text` parts, added `label`/`fontSize` to `Placement`, rendered text in `PlacementNode`. Build passed but approach was wrong — text shouldn't depend on parts.

2. **Refactored — first-class `TextPlacement`:** Reverted part-based approach. Added `TextPlacement` interface and `texts[]` to `Layer`. Created `TextNode` component with selection, drag, and per-layer rendering (`#fff` for silkscreen, `#c07a3a` for copper). Added `addTextPlacement`, `removeTextPlacement`, `updateTextPlacement` store actions with undo support. Properties panel detects text selection and shows text/font-size/position/rotation/locked fields. Delete handler covers both placements and texts.

**Files changed:** `core/types/panel.ts`, `core/types/index.ts`, `core/types/part.ts`, `core/sample-parts.ts`, `stores/panelStore.ts`, `stores/uiStore.ts`, `components/Editor/TextNode.tsx` (new), `components/Editor/PanelCanvas.tsx`, `components/Editor/PlacementGhost.tsx`, `components/Editor/PlacementNode.tsx`, `components/Toolbar/Toolbar.tsx`, `components/PropertiesPanel/PropertiesPanel.tsx`.

## Problem Areas

1. **Copper text on aluminium panel** — now blocked, but existing copper text placements on aluminium panels would be stale. No migration logic.
2. **`activeLayerId` still unused for single-part placement** — only text uses it. Parts still go to their intrinsic `part.layerType`.
3. **No check for existing text placements when deleting copper text on aluminium** — if material is changed to aluminium, existing copper texts are orphaned (not rendered but still in data).
4. **`updatePart` still doesn't cascade to placements** — stale data risk when part dimensions/name/layerType change.
